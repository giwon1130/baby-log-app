# 카피 가이드

> **누가 봄**: 디자이너 / FE / PM
> **언제 봄**: UI 텍스트 작성/리뷰

## 톤

**친근한 반말체.** 부담 안 주고 옆에서 도와주는 친구 느낌.

✅ "다음부터 더 정확해질 거예요 👶"
❌ "데이터를 학습하여 정확도를 향상시키겠습니다."

## 핵심 원칙

### 1. 의학적 단정 금지
이 앱은 의료기기 아님. 모든 분석/조언에 "참고용" 뉘앙스 유지.

✅ "통증 신호일 수 있어요"
✅ "배고픔 패턴 가능성"
❌ "통증입니다"
❌ "지금 수유해야 합니다"

### 2. 사용자 잘못 분위기 금지

✅ "분석에 실패했어요. 다시 시도해주세요"
✅ "마이크 권한이 필요해요"
❌ "잘못된 입력입니다"
❌ "권한을 허용하지 않으셨습니다"

### 3. 진행 상태는 동작형으로

✅ "듣는 중...", "분석 중..."
❌ "녹음 시작됨", "분석 진행 상태"

### 4. 숫자 단위 명시

✅ "음 높이 720 Hz", "리듬성 0.62"
❌ "음 높이 720", "0.62"

### 5. 시간 표기

- 같은 날: `오늘 14:30`
- 어제: `어제 14:30`
- 그 외: `4/25 14:30`

## 표준 표현

| 상황 | 사용 표현 |
|------|-----------|
| 데이터 없음 | "아직 ...이 없어요" |
| 로딩 | "..." (단순 ActivityIndicator) 또는 "X 중..." |
| 에러 (일반) | "잠시 후 다시 시도해주세요" |
| 에러 (네트워크) | "네트워크를 확인해주세요" (P2 — 현재 raw error) |
| 저장 완료 | "저장됐어요" |
| 권한 필요 | "X 권한이 필요해요" |
| 다음 단계 유도 | "...해주시면 ..." |
| 학습 진행 | "다음부터 더 정확해져요" |

## 라벨 한국어 (절대 변경 금지 — DB enum과 매핑)

| 코드 | 표시 |
|------|------|
| HUNGER | 배고픔 |
| TIRED | 졸림 |
| DISCOMFORT | 불편함 |
| BURP | 트림 필요 |
| PAIN | 통증 |
| UNKNOWN | 모르겠음 |

## 이모지 사용

절제. 결과 안내 / 학습 진행 같은 긍정 메시지에만 1개.

✅ "👶" (아기 / 학습 진행 시)
✅ "✨" (학습 단계 배너 — 아이콘으로 대체했지만 OK)
❌ "🔔🔔🔔 알림 왔어요!"

## 알림 (로컬 푸시)

| 종류 | 제목 | 본문 |
|------|------|------|
| 다음 수유 5분 전 | "곧 수유 시간이에요" | "마지막 수유 후 N시간 N분 지났어요" |

## 가이드 미준수 시

코드 리뷰에서 카피 코멘트는 "WORDING:" 프리픽스로 표시.
변경 시 PR 설명에 영향 화면 명시.
