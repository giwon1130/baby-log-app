- 외부 사용자 50명 이상
- 회귀 사고 (이미 동작했던 기능이 깨짐)

## 수동 테스트 시나리오 (릴리스 전 체크리스트)

### Smoke (5분)
- [ ] 앱 cold start → FamilySetup 또는 Main 정상 진입
- [ ] 가족 코드 발급 → 다른 기기에서 합류 → 같은 데이터 보임
- [ ] 수유 기록 → 홈에 다음 수유 시각 표시 → 알림 등록 확인 (아기 탭 → 알림 설정)
- [ ] 기저귀/수면/성장 각각 1건 기록
- [ ] 통계 탭 — 차트 정상 표시

### 울음 분석 (Phase 2A 기준)
- [ ] 마이크 권한 첫 요청 시 안내 → 허용 → 녹음 5초
- [ ] 결과 화면: 라벨 + 신뢰도 + 분석 디테일 + 다른 가능성 표시
- [ ] "맞았어요" → 학습 단계 confirmed 카운트 +1
- [ ] "다른 이유예요" → 모달 → 6종 라벨 → 선택 → 정정 반영
- [ ] 분석 기록 보기 → 통계 카드 + 시간순 리스트
- [ ] 미확인 카드 탭 → 정정 모달 → 라벨 → 통계 즉시 갱신

### 백엔드 (curl)
```bash
# 가족 생성
curl -X POST $API/api/v1/families
# → { id, inviteCode }

# 아기 생성
curl -X POST $API/api/v1/families/$FID/babies \
  -H "Content-Type: application/json" \
  -d '{"name":"테스트","birthDate":"2026-04-01","gender":"FEMALE"}'

# 울음 샘플 제출
curl -X POST $API/api/v1/babies/$BID/cry-samples \
  -H "Content-Type: application/json" \
  -d '{"durationSec":5,"pitchMeanHz":650,"rhythmicity":0.5}'
```

응답 라벨이 합리적인지 (피치 650 → PAIN 가능성, 리듬 0.5 → HUNGER 가능성).
