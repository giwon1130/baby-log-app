# 데이터 모델 (앱 타입)

> **누가 봄**: FE
> **언제 봄**: 새 도메인 추가, 응답 구조 확인
> **단일 진실의 원천**: [API 저장소 데이터모델](https://github.com/giwon1130/baby-log-api/tree/main/docs/엔지니어링/데이터모델.md)

이 문서는 FE에서 쓰는 **TypeScript 타입** 위주. DB 스키마와 조인은 API 저장소가 SoT.

## 울음 분석

### CryLabel
```ts
type CryLabel = 'HUNGER' | 'TIRED' | 'DISCOMFORT' | 'BURP' | 'PAIN' | 'UNKNOWN'
```

### CryPrediction
```ts
type CryPrediction = {
  label: CryLabel
  labelDisplay: string       // "배고픔" 등 한국어
  confidence: number         // [0, 1]
  reasons: string[]          // "마지막 수유 후 3시간 경과" 등
}
```

### CryLearningStage
```ts
type CryLearningStage = {
  confirmedCount: number
  stage: 'HEURISTIC' | 'SIMILARITY' | 'PERSONAL'
  stageDisplay: string
  nextStageAt: number | null
  nextStageDisplay: string | null
}
```

### CrySample (전체)
```ts
type CrySample = {
  id: string
  babyId: string
  recordedAt: string
  durationSec: number
  predictions: CryPrediction[]      // 점수 내림차순
  confirmedLabel: CryLabel | null
  confirmedLabelDisplay: string | null
  learningStage: CryLearningStage
  note: string
}
```

> ⚠️ 응답에는 음향 feature 원본 값(피치/리듬 등) 안 포함. 원본은 DB에만. 클라이언트
> 화면에서 보여주려면 `recordAndAnalyze()` 응답을 같이 들고 있어야 함 (현재
> CryMonitorScreen이 `lastFeatures` state로 보관).
>
> Phase 2B에서 history 진입 시 디테일 보고 싶으면 응답에 추가 검토.

### CryFeatureSummary (네이티브 모듈 응답)
```ts
type CryFeatureSummary = {
  durationSec: number
  cryConfidenceAvg: number | null
  cryConfidenceMax: number | null
  avgVolumeDb: number | null
  peakVolumeDb: number | null
  sampleCount: number
  // Phase 2A 추가
  pitchMeanHz: number | null
  pitchStdHz: number | null
  pitchMaxHz: number | null
  voicedRatio: number | null
  zcrMean: number | null
  rhythmicity: number | null
}
```

각 feature 의미는 [`제품/용어집.md`](../제품/용어집.md) "음향 feature" 표 참고.

## 명명 규칙

- 시각: `xxxAt` (`fedAt`, `changedAt`, `sleptAt`, `wokeAt`)
- 식별자: `xxxId`
- 단위 명시: `amountMl`, `weightG`, `heightCm`, `pitchMeanHz`, `volumeDb`
- enum: SCREAMING_SNAKE_CASE 문자열 (`HUNGER`, `WET`)
- nullable: 명시적으로 `| null`

## 응답 변형 시 영향

타입 바뀌면:
1. `src/types/index.ts` 갱신
2. 사용하는 화면/컴포넌트 컴파일 에러 따라 수정
3. 백엔드 같이 — `baby-log-api` 의 `data class` 변경 + 컬럼 마이그레이션
