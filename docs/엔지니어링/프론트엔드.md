# 프론트엔드 가이드

> **누가 봄**: FE 합류자 / 본인 (잊어버렸을 때)
> **언제 봄**: 새 화면 만들 때, 빌드 막힐 때

## 환경

- macOS (Apple Silicon — Xcode 필수)
- Node 20+ (Volta 권장)
- Xcode 15+ + iOS 17+ SDK
- CocoaPods (sudo 없이: `gem install --user-install cocoapods` 또는 brew)
- Apple Developer 무료 계정 (실기기 빌드용)

## 실행 흐름

### 1. JS만 수정 → Metro 핫리로드

```bash
cd baby-log-app
npx expo start --port 8081
```

아이폰 흔들기 → "Reload" 또는 자동 reload (저장 시).

**전제**:
- 아이폰 ↔ 맥이 같은 WiFi
- 8081 포트가 이 프로젝트로 점유됨 (다른 Expo 프로젝트가 점유 중이면 충돌 — 아래 함정)

### 2. 네이티브(Swift / Pod / Info.plist) 수정 → Xcode 재빌드 필수

```bash
cd ios
pod install                                          # Podfile 변경 시만

xcodebuild -workspace BabyLog.xcworkspace -scheme BabyLog \
  -configuration Debug \
  -destination "id=$DEVICE_UDID" \
  -derivedDataPath build \
  CODE_SIGN_STYLE=Automatic DEVELOPMENT_TEAM=776H9NV6HT \
  -allowProvisioningUpdates build

xcrun devicectl device install app --device $DEVICE_UDID \
  build/Build/Products/Debug-iphoneos/BabyLog.app
```

UDID 확인: `xcrun devicectl list devices`

## 함정 / 사고 학습

[`운영/장애대응.md`](../운영/장애대응.md) 와 중복 — 그쪽이 SoT.

핵심만:
- **"RNGoogleSignin not found"** → 다른 프로젝트(otaku-feed) Metro가 8081 점유. `lsof -i :8081`로 확인 후 kill
- **"No script URL provided"** → Metro 죽었음. 백그라운드로 다시 띄우려면 `nohup ... & disown`
- **Pod CocoaPods Unicode 에러** → `export LANG=en_US.UTF-8 LC_ALL=en_US.UTF-8`
- **Linker `Sealable::Sealable() undefined`** → Pod 캐시 오염. `pod deintegrate && rm -rf Pods Podfile.lock build && pod install`

### 화면 분리 가이드 (300줄 넘으면)
- 한 화면이 여러 영역 (헤더/리스트/모달) 가지면 → `src/components/<도메인>/` 로 분리
- pure 헬퍼는 `src/utils/` 로
- 사례: `CryMonitorScreen.tsx` 595줄 → 287줄 (5개 컴포넌트로 분리)

### 새 네이티브 모듈
1. `modules/<name>/` 폴더 생성
2. `expo-module.config.json` (Expo가 autolink)
3. `package.json`, `ios/<Name>Module.podspec`, `ios/<Name>Module.swift`
4. JS 브리지: `index.ts` 에 `requireNativeModule` + 타입
5. `cd ios && pod install` → 빌드

기존 모듈 참고: [`modules/cry-detector/`](../../modules/cry-detector)

## 상태 관리

전역 상태 라이브러리 안 씀 (Redux/Zustand 등). 각 화면이 자체 useState + useEffect로
fetch. 화면 간 공유 데이터는 AsyncStorage (`familyId`, `babyId`).

이유: 작은 앱이고, 화면 진입 시 fresh fetch 하는 패턴이 자연스러움.

상태 라이브러리 도입 검토 시점: 같은 데이터를 3+ 화면이 공유하면서 동기화 필요 시
(예: 실시간 수면 진행 표시).

## API 호출 컨벤션

```ts
// babyLogApi.ts
export const recordFeed = (babyId: string, data: { ... }) =>
  api.post<FeedRecord>(`/api/v1/babies/${babyId}/feeds`, data)
```

- 도메인별 함수 export
- 응답 타입 명시 (`<FeedRecord>`)
- 컴포넌트에서 try/catch + 사용자 친화 에러 메시지 ([`디자인/카피가이드.md`](../디자인/카피가이드.md))

## 성능 주의

- `ScrollView` 안에 큰 리스트 넣지 말고 `FlatList` 사용 (현재 거의 ScrollView 쓰는데, 100+ 항목 화면이면 변경)
- 이미지 거의 없음 (현재). 추가 시 `expo-image` 검토
- 차트는 `react-native-chart-kit` — 데이터 100건 이상이면 느림. sampling 검토
