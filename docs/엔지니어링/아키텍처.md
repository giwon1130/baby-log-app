## 데이터 흐름 — 일반 기록

```
사용자 탭
  → Screen 컴포넌트의 onPress
  → babyLogApi.recordFeed(babyId, {...})
  → axios POST /api/v1/babies/:id/feeds
  → Spring 서비스 → JdbcTemplate INSERT
  → 응답 (FeedRecord)
  → React state 업데이트 → 화면 리렌더
```

## 데이터 흐름 — 울음 분석 (가장 복잡)

```
사용자 "분석 시작" 탭
  → CryDetector.recordAndAnalyze(5)            # JS → 네이티브
    └─ Swift AVAudioEngine 시작
       AVAudioPCMBuffer 콜백 → 3개 처리:
         ├─ SNAudioStreamAnalyzer (infant_cry 신뢰도)
         ├─ VolumeMeter (RMS dB)
         └─ AudioFeatureExtractor (피치/ZCR/리듬성, vDSP 사용)
       5초 후 Promise resolve { durationSec, pitchMeanHz, ... }
  → submitCrySample(babyId, features)          # 숫자만 전송
  → Spring CryAnalysisService.classify():
       ├─ buildContext(babyId)                  # DB 조회: 마지막 수유/기저귀/수면
       ├─ apply prior + 음향 규칙 → 라벨별 점수
       ├─ if 확정샘플 ≥20: similarity 부스트
       └─ insert bl_cry_samples (예측 + feature)
  → 응답 CrySampleResponse
  → ResultView 렌더 (예측 + 디테일 + 정정 버튼)
사용자 "맞았어요" 또는 "다른 이유"
  → confirmCrySample(sampleId, label)
  → PATCH /api/v1/cry-samples/:id/confirm
  → DB UPDATE confirmed_label
  → 학습 단계 자동 승급 (확정 카운트 기반)
```

## 핵심 설계 결정 (요약)

자세한 건 [`의사결정로그.md`](의사결정로그.md):

1. **온디바이스 feature 추출** — 음성 파일 업로드 안 함 (프라이버시 + 비용)
2. **Expo 로컬 모듈** — react-native-link 안 쓰고 자체 네이티브 패키지화
3. **JdbcTemplate** — JPA 안 씀 (SQL 명시적, 마이그레이션 단순)
4. **Stack + Tab 분리** — 모달성 화면(GrowthRecord, CryHistory)은 Stack 푸시
5. **CryAnalysisService 모놀리식** — 분류기 규칙 너무 많아지면 데이터화 (Phase 2B)

## 외부 의존성

### 클라이언트
- `expo` (54)
- `react-navigation/{native,bottom-tabs,native-stack}`
- `axios`
- `expo-notifications` — 로컬 알림
- `react-native-chart-kit` — 통계 차트
- `expo-modules-core` — 자체 네이티브 모듈
- iOS only: `SoundAnalysis`, `AVFoundation`, `Accelerate` (시스템 프레임워크)

### 서버
- Spring Boot 3.3 / Kotlin 1.9
- `spring-jdbc`, `postgresql` 드라이버
- (향후) `spring-security` — 인증 도입 시점

## 미해결 / 알려진 한계

- **인증 없음** — 가족 코드만으로 접근. 코드 추측당하면 데이터 노출. [`운영/보안.md`](../운영/보안.md)
- **오프라인 동작 없음** — 네트워크 끊기면 기록 불가
- **다중 아기 UI** — 백엔드는 지원하지만 프론트는 첫 아기만 사용
- **테스트 커버리지** — 자동 테스트 거의 없음. 수동 시나리오만 [`품질/테스트전략.md`](../품질/테스트전략.md)
