# 환경변수 (앱)

> **누가 봄**: 본인 / DevOps
> **언제 봄**: 새 환경 셋업, 빌드 제대로 안 될 때
> **단일 진실의 원천**: 서버는 [API 저장소](https://github.com/giwon1130/baby-log-api/tree/main/docs/운영/환경변수.md)

## 클라이언트

### `BABY_LOG_API_URL`
- **용도**: API 베이스 URL
- **기본값**: `app.config.ts` 에 박힌 값 (현재 Railway 도메인)
- **로컬 테스트**: `BABY_LOG_API_URL=http://<맥미니IP>:8092 npx expo start`
- **위치**: `app.config.ts` → `extra.apiUrl` → `client.ts` 가 읽음

```bash
# 로컬 백엔드 띄우고 실기기에서 접속
BABY_LOG_API_URL=http://192.168.1.10:8092 npx expo start --port 8081
```

> ⚠️ **iOS 시뮬레이터**: `localhost` 가능
> ⚠️ **실기기**: 반드시 맥 IP. `localhost` 안 됨 (디바이스 자기 자신을 가리킴)

### EAS 빌드 시 환경변수

`eas.json` 의 `env` 키에 빌드 프로파일별로 명시:
```json
{
  "build": {
    "production": {
      "env": {
        "BABY_LOG_API_URL": "https://baby-log-api.up.railway.app"
      }
    }
  }
}
```

### Apple 관련 (코드 외부)

| 변수 | 어디 | 용도 |
|------|------|------|
| `DEVELOPMENT_TEAM` | xcodebuild 명령 | 본인 팀 ID `776H9NV6HT` (자동 서명) |
| App Store Connect API Key | EAS / 로컬 | `eas submit` 자동 제출 시 |
| APNs Auth Key | App Store Connect → Keys | 푸시 알림 |

이런 비밀값은 git에 **절대** 안 들어감. EAS / Apple Developer 계정에 저장.

## 미사용 / 향후

- `SENTRY_DSN` — 에러 추적 도입 시 (P2)
- `MIXPANEL_TOKEN` — 사용성 분석 (P3)

## 동작 검증

```bash
# 빌드 후 실행하고 dev menu에서 "Show Dev Settings" → "Bundler URL"
# 또는 컴파일된 앱이 실제로 어디로 호출하는지 확인
npx expo config --type prebuild | grep -A2 apiUrl
```

## 트러블슈팅

| 증상 | 원인 | 해결 |
|------|------|------|
| "Network Error" 즉시 발생 | API URL 오타 또는 도달 불가 | `curl $URL/api/v1/families/test` 로 검증 |
| 시뮬레이터는 되는데 실기기 안 됨 | `localhost` 박힘 | 맥 IP로 교체 후 재빌드 |
| 빌드는 됐는데 운영 URL로 안 감 | EAS env 설정 누락 | `eas.json` 의 `env` 확인 |
