## 데이터 분류

| 카테고리 | 예시 | 보호 수준 |
|---------|------|-----------|
| 식별 정보 | 아기 이름 | **중간** — 닉네임 권장 |
| 행동 로그 | 수유/기저귀/수면 시각·양 | **중간** |
| 음성 데이터 | 울음 녹음 | **최상** — 기기 외 미전송 (특별 보호) |
| 음향 feature | 피치, 음량 등 숫자 | **낮음** — 음성 복원 불가 |

## 음성 데이터 처리 정책 (가장 중요)

**원칙**: **음성 파일은 기기에서도 저장하지 않는다.**

구현:
- iOS `AVAudioPCMBuffer` → `SNAudioStreamAnalyzer` + `AudioFeatureExtractor` 처리
- 파일 시스템에 쓰는 코드 **없음** (AVAudioFile 미사용)
- 메모리 버퍼는 분석 끝나면 해제
- 서버로는 숫자 feature만 (`pitchMeanHz`, `rhythmicity` 등)

검증 방법:
```swift
// CryDetectorModule.swift 에 AVAudioFile 검색 → 결과 없어야 함
```

## 인증 / 권한 (현재)

**현재**: **인증 없음.** 가족 초대 코드 8자리만으로 모든 기록 접근 가능.

- API endpoint는 `babyId`만 알면 호출 가능
- 초대 코드는 영숫자 8자리 — 약 2.8조 경우의 수, 무차별 추측 비현실적이지만 보장 X

**리스크**:
1. 코드 추측 (미미함)
2. 코드 공유 후 회수 불가 (관계 끊긴 가족 멤버가 계속 접근)
3. 클라이언트 트래픽 가로채기 — 인증 없으니 추가 검증 없음

**완화 (현재)**:
- HTTPS 강제 (Railway 자동)
- 초대 코드 길이 충분
- 로그에 PII (이름) 안 찍기

## 인증 도입 트리거

다음 중 하나라도 발생 시 즉시 도입:
- 외부 사용자 100명 초과
- 가족 멤버 회수 / 추방 기능 요구 발생
- 데이터 유출 사고
- 인앱결제 도입 (계정 필요)

도입 옵션:
- (A) Firebase Auth — Apple ID / Google 로그인. 빠름.
- (B) 자체 JWT — 백엔드 Spring Security 추가
- (C) Apple Sign-In + 자체 백엔드 토큰

권장: **(A)** Firebase. 1인 운영에 적합.

## App Store Connect 공개 항목

심사 시 정확히 명시:

| App Privacy 항목 | 답변 |
|-----------------|------|
| 사용 데이터 수집 | YES — 사용 데이터 (서버 로그) |
| 식별자 수집 | YES — `familyId` (자체 발급, Apple ID 아님) |
| 음성/오디오 수집 | **NO — 분석은 기기 내에서만** |
| 건강 정보 수집 | YES — 수유/수면/체중 (사용자 직접 입력) |
| 위치 / 광고 ID | NO |
| 제3자 SDK 데이터 공유 | NO (현재 분석 SDK 미사용) |

## 마이크 권한 문구

`app.config.ts` → `NSMicrophoneUsageDescription`:
```
"아기 울음을 분석해 어떤 이유로 우는지 추정하는 데에만 사용됩니다.
음성은 기기를 떠나지 않습니다."
```

심사 통과 핵심 요건:
- 사용 이유 구체적 ("울음 분석")
- 데이터 비전송 명시 ("기기를 떠나지 않습니다")
- 모호한 표현 ("앱 사용성 향상" 등) 금지
