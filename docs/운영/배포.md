# 배포 (앱)

> **누가 봄**: 본인 / DevOps
> **언제 봄**: 새 빌드 만들 때, 배포 막힐 때
> **함께 봐야 할**: [API 저장소 배포](https://github.com/giwon1130/baby-log-api/tree/main/docs/운영/배포.md) — 백엔드 자동 배포 (Railway)

## 배포 매트릭스

| 채널 | 대상 | 인증 | 자동/수동 | 빈도 |
|------|------|------|----------|------|
| **로컬 개발** | 본인 iPhone | Apple Dev (free or paid) | 수동 | 매일 |
| **TestFlight** | 가족/지인/QA | Apple Dev **유료** | 수동 → Auto Submit | 주 1회 (목표) |
| **App Store** | 공개 | Apple Dev 유료 + 심사 | 수동 | 분기 1회 |

> 2026-04 기준 유료 승인 완료 → TestFlight 진입 가능.

## 1. 로컬 개발 빌드 → 본인 iPhone

전제: 디바이스 페어링 1회 완료, Wi-Fi 같은 망.

```bash
cd baby-log-app/ios

# 의존성 (Pod 변경 있을 때만)
export LANG=en_US.UTF-8 LC_ALL=en_US.UTF-8
pod install

# 빌드
xcodebuild -workspace BabyLog.xcworkspace -scheme BabyLog \
  -configuration Debug \
  -destination "id=$DEVICE_UDID" \
  -derivedDataPath build \
  CODE_SIGN_STYLE=Automatic DEVELOPMENT_TEAM=776H9NV6HT \
  -allowProvisioningUpdates build

# 설치
xcrun devicectl device install app --device $DEVICE_UDID \
  build/Build/Products/Debug-iphoneos/BabyLog.app
```

UDID 조회: `xcrun devicectl list devices`

처음 설치 후 아이폰에서: 설정 → 일반 → VPN 및 기기 관리 → 개발자 앱 신뢰.

JS만 바꿨으면 빌드 다시 안 해도 됨 — Metro reload (앱 흔들기).

## 2. TestFlight 베타

> 유료 승인 직후 첫 진입 절차. EAS (Expo Application Services) 사용.

### 2-1. 첫 셋업 (1회)
1. App Store Connect (https://appstoreconnect.apple.com) → "내 앱" → 신규 앱
   - 번들 ID: `com.giwon.babylog`
   - 이름: 베이비로그 (또는 BabyLog)
   - 카테고리: Health & Fitness 또는 Lifestyle
2. EAS 로그인:
   ```bash
   npx eas login          # Expo 계정
   ```
3. `eas.json` 확인 — 이미 있음. 빌드 프로파일 `production` 정의.

### 2-2. 빌드 + 제출

```bash
# iOS 빌드 (EAS 클라우드에서 실행 — 맥 빌드 필요 없음)
npx eas build --platform ios --profile production

# 빌드 끝나면 .ipa URL 받음
# 자동 제출:
npx eas submit --platform ios --latest
```

### 2-3. TestFlight 그룹
- **Internal**: Apple ID 등록자, 즉시 사용 (심사 X, 100명 제한)
- **External**: 베타 테스트 검토 1회 통과 후 외부 사용자 (10,000명까지)

### 2-4. 알려진 함정
- **Push 자격증명**: APNs key 발급 (Identifiers → Keys → APNs) 후 EAS에 업로드
- **Privacy Manifest**: 새 정책 — `app.config.ts` 의 `privacyManifests` 추가 필요할 수 있음
- **Provisioning 자동 생성**: EAS가 자동 처리. 수동 매니징 X

## 3. App Store 정식 출시

TestFlight에서 **2주 이상** 안정성 확인 후.

### 체크리스트
- [ ] 스크린샷 (iPhone 6.7" / 5.5") 6장 이상
- [ ] 앱 설명 (한국어, 4000자 이내)
- [ ] 키워드 (100자)
- [ ] 개인정보처리방침 URL — [`보안.md`](보안.md) 의 정책 따로 호스팅
- [ ] 지원 URL (이메일 OK)
- [ ] 카테고리 + 연령 등급 (4+)
- [ ] App Privacy 질문지 (음성 데이터 처리 정확히 명시)

### 심사 거절 유발 리스크
1. **마이크 권한** — 사용 이유 명시 부족 시 거절. 현재 `NSMicrophoneUsageDescription`
   문구: "아기 울음을 분석해 어떤 이유로 우는지 추정하는 데에만 사용됩니다. 음성은
   기기를 떠나지 않습니다." (확실히 "기기 외 미전송"명시)
2. **의료 조언** — "통증입니다"라고 단정하면 거절. [`디자인/카피가이드.md`](../디자인/카피가이드.md)
3. **의미 없는 페이지** — 빈 화면 / placeholder 있으면 거절
