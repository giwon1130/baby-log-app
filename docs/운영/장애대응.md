# 장애 대응 (Runbook)

> **누가 봄**: 본인 / 합류자
> **언제 봄**: 사고 발생 시. 같은 사고 재발 방지를 위해 모든 사고는 여기 추가.

## 빌드 / 개발 환경 사고

### Metro 8081 포트가 다른 프로젝트에 점유됨
**증상**: 앱 실행 시 `RNGoogleSignin not found` 또는 엉뚱한 화면.
**원인**: 다른 Expo 프로젝트(예: otaku-feed)가 8081에서 돌고 있어서 거기 번들 받아옴.
**해결**:
```bash
lsof -i :8081
kill <PID>
cd baby-log-app && npx expo start --port 8081
```
**예방**: 동시 개발 시 다른 포트 (`--port 8082`) 사용. 단 앱 빌드 시 박힌 포트와 일치 필요.

### CocoaPods Unicode 에러
**증상**: `pod install` 시 `Encoding::CompatibilityError` / `UnicodeNormalize.normalize`.
**원인**: 시스템 locale 미설정.
**해결**:
```bash
export LANG=en_US.UTF-8 LC_ALL=en_US.UTF-8
pod install
```
`.zshrc` / `.bashrc` 에 영구 추가 권장.

### Pod 캐시 오염 (linker 에러)
**증상**: `Undefined symbol: facebook::react::Sealable::Sealable()` 또는 비슷한 RN
내부 심볼 미해결.
**원인**: Pod 변경 후 DerivedData가 옛날 버전 캐싱.
**해결**:
```bash
cd ios
pod deintegrate
rm -rf Pods Podfile.lock build
pod install
xcodebuild ... clean build
```

### `expo run:ios` 가 멈춘 듯 보임
**원인**: stdout 버퍼링 (특히 `... | tail -20` 같은 파이프).
**해결**: 백그라운드로 띄우고 직접 빌드 + `devicectl install`:
```bash
nohup npx expo start > /tmp/metro.log 2>&1 & disown
xcodebuild ... build
xcrun devicectl device install app ...
```

### "No script URL provided" on iPhone
**원인**: Metro 죽음 (백그라운드 task timeout 등).
**해결**: Metro 다시 띄우기:
```bash
nohup npx expo start --port 8081 > /tmp/metro-babylog.log 2>&1 & disown
```
앱 재시작 또는 dev menu → Reload.

### Pod 서명 에러 (Pods.xcodeproj)
**증상**: `Signing for "EXConstants-..." requires a development team`
**원인**: xcodebuild 명령에 `DEVELOPMENT_TEAM` 미지정.
**해결**:
```bash
xcodebuild ... CODE_SIGN_STYLE=Automatic DEVELOPMENT_TEAM=776H9NV6HT \
  -allowProvisioningUpdates build
```

### Metro 캐시 stale (새 컴포넌트 안 보임)
**증상**: 코드 추가했는데 앱에서 안 보임. `curl /index.bundle | grep <NewSymbol>` 결과 없음.
**해결**: Metro 재시작 with `--clear`:
```bash
kill <metro_pid>
npx expo start --clear --port 8081
```

## 런타임 / 운영 사고

### 마이크 권한 거부됨
**증상**: 울음 분석 시작 시 "마이크 권한이 필요해요" 에러.
**해결 (사용자)**: iOS 설정 → 베이비로그 → 마이크 토글 ON.
**예방 (앱)**: 권한 거부 시 설정 앱으로 보내는 deep link (`Linking.openSettings()`) 추가 — 현재 P2.

### 백엔드 응답 느림 (5초+)
**원인 1**: Railway 무료 티어 cold start.
**해결**: 첫 요청 후 warmup 됨. 사용자에겐 그냥 로딩 표시.
**원인 2**: PostgreSQL 응답 느림.
**해결**: Railway 대시보드 → DB 메트릭. 인덱스 누락이면 추가.

### 가족 코드로 합류했는데 데이터 안 보임
**원인**: babyId AsyncStorage 미저장 (코드 버그).
**해결**: 앱 설정 → 데이터 삭제 → 다시 가족 코드로 합류.
**예방**: `joinFamily` 후 `setStoredFamilyId` + 첫 baby fetch + `setStoredBabyId` 시퀀스 검증.
