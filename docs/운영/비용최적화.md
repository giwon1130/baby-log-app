## 트리거 — 비용 증가 예상 시점

### Railway 유료 전환 ($5/월 → $20+/월)
- 일 활성 사용자 100명 초과 추정 시
- DB 1GB 초과
- API 응답 시간 무료 cold start 때문에 거슬리기 시작

### 외부 ML 도입 시
- Phase 2B에서 YAMNet 임베딩이 서버 추론으로 전환되면 GPU 비용 발생
- 현재 계획은 YAMNet 도 클라이언트 (CoreML) → **추가 비용 0**

### CDN / 이미지 호스팅
- 향후 아기 사진 첨부 기능 도입 시 (P3)
- S3 + CloudFront or Cloudflare R2 검토

### Push 알림
- APNs 자체는 무료
- 백엔드에서 푸시 발송하면 추가 트래픽 — Railway 무료 티어 내 충분

## 비용 절감 원칙

1. **온디바이스 우선** — ML 추론은 가능한 한 클라이언트에서
2. **무료 티어 한계 모니터링** — Railway 사용량 대시보드 주간 확인
3. **유료 SDK 도입 신중** — Sentry 등도 사용자 100명 넘으면 도입
4. **이미지/동영상 회피** — 베이비로그 본질에 사진 필수 아님

## 모니터링

- Railway 대시보드 → Usage → 월간 크레딧 소진 예상치
- App Store Connect → Sales & Trends (다운로드 수)
- (향후) 자체 분석: API 호출 빈도, 사용자당 평균 기록 수
