# 비전

> **누가 봄**: PM / 합류하는 모든 역할
> **언제 봄**: 새 기능 우선순위 정할 때, "이거 만들어야 하나?" 의심 들 때

## 한 줄 소개

**신생아 부모를 위한 통합 기록 + AI 울음 분석 앱.** 수유/기저귀/수면/성장 기록을
한 곳에서 가족과 공유하고, 우는 이유를 학습형 AI로 알려준다.

## 누구를 위해

- **1차**: 신혼부부 (생후 0~12개월 첫 아기 키우는 부부)
  - 둘 다 기록 안 놓치고 동기화하고 싶음
  - 우는 이유 매번 추측만 하고 정답을 모름
  - 산후조리원 / 시댁 / 친정 등 손이 자주 바뀜 → 일관된 기록 필요
- **2차**: 시댁/친정 어른들 — 같은 가족 코드로 들어와서 보기

## 왜 만들어야 하나

기존 앱들의 문제:
1. **기록만 — 분석/조언 없음** (베이비타임, 베이비 트래커)
2. **분석은 있지만 비싸고 영어** (ChatterBaby — $9.99/mo, 미국)
3. **가족 공유 안 됨 또는 어색함** (혼자 폰 들고 다님)

베이비로그가 다른 점:
- **무료** + **한국어** + **가족 공유 기본**
- **온디바이스 울음 분석** (음성 파일 업로드 안 함 — 프라이버시)
- **개인화 학습** — 우리 애 울음에 점점 맞춰짐

## KPI (어떻게 잘 되는지 측정)

> ⚠️ 아직 출시 전이라 가설. 출시 후 보정.

| 단기 (출시 첫 3개월) | 목표 |
|----------------------|------|
| MAU | 100명 (지인 + 입소문) |
| 일평균 기록 횟수 | 10회/사용자 (기록만으로도 가치) |
| 울음 분석 사용자 비율 | 30% (옵션이라 강요 아님) |
| 분석 후 라벨 확정 비율 | 50% (학습 루프가 돌아가는 신호) |

| 중기 (6~12개월) | 목표 |
|----------------|------|
| 가족당 평균 기기 수 | 1.5+ (공유가 의미 있다는 증거) |
| 울음 분석 30회+ 가족 비율 | 30% (개인화 단계 진입) |
| 정정 후 재예측 정확도 향상 | 측정 가능한 +10%p |

## 비-목표 (안 하는 것들)

- **수면 컨설팅 / 의학적 진단** — 위험. "참고용입니다" 명시
- **바이탈 사인 추적** (체온/맥박) — 의료기기 영역
- **광고 / 인앱결제** — 일단 무료로 신뢰 확보 우선
- **육아 커뮤니티 / 게시판** — 다른 앱들이 잘함. 베이비로그는 도구에 집중
- **다국어** — 1차는 한국어만. 시장 검증 후 결정
