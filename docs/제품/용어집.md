# 용어집

> **누가 봄**: 모두 (PM/디자이너/FE/BE/QA)
> **언제 봄**: 처음 합류할 때, 새 기능 명명할 때
> **원칙**: 코드 / UI 텍스트 / 문서 모두 이 표의 용어를 그대로 사용

용어가 흔들리면 사용자도 혼란스럽고 코드 검색도 안 됨. 새 용어 도입 시 여기 먼저 추가.

## 도메인

| 한국어 | 영어 (코드) | UI 표시 | 정의 |
|--------|-------------|---------|------|
| 가족 | `Family` | 가족 | 한 아기를 함께 돌보는 사람들의 그룹. 초대 코드로 합류 |
| 초대 코드 | `inviteCode` | 초대 코드 | 8자리 영숫자. 가족 합류 시 입력 |
| 아기 | `Baby` | 아기 | 가족에 속한 기록 대상. 한 가족에 여러 명 가능 |
| 일령 | `daysOld` | N일째 | 출생 후 경과 일수. UI에서 "생후 N일"로 표기 |
| 수유 | `Feed` | 수유 | 모유/분유/혼합. 양(ml) 또는 좌/우 시간(분) |
| 기저귀 | `Diaper` | 기저귀 | WET/DIRTY/MIXED/DRY |
| 수면 | `Sleep` | 수면 / 잠 | 시작과 종료가 분리된 세션. 진행 중이면 `wokeAt = null` |
| 성장 기록 | `GrowthRecord` | 성장 기록 | 체중/키/머리둘레 측정값 |
| 성장 단계 | `GrowthStage` | 성장 단계 | 일령 기반 자동 산정 (신생아기/영아기 등) |

## 울음 분석

| 한국어 | 영어 (코드) | UI 표시 | 정의 |
|--------|-------------|---------|------|
| 울음 샘플 | `CrySample` | 분석 기록 | 한 번의 녹음 + 추출된 feature + 예측 결과 |
| 라벨 | `CryLabel` | 이유 | 울음의 추정 이유. 6종 enum |
| 예측 | `CryPrediction` | 가능성 | 라벨별 점수 + 근거 reason 리스트 |
| 음향 feature | `acoustic features` | (디테일) | 피치, ZCR, 리듬성 등 — 이 표 아래 별도 표 |
| 학습 단계 | `LearningStage` | 학습 단계 | HEURISTIC / SIMILARITY / PERSONAL — 확정 샘플 수에 따라 자동 승급 |
| 컨텍스트 | `CryContext` | (내부) | 마지막 수유/기저귀/수면 경과 + 일령 + 시각 |

### 라벨 enum

| 코드 | UI 한국어 | 의미 |
|------|----------|------|
| `HUNGER` | 배고픔 | 수유 시간 도래 / 입맛 다시기 |
| `TIRED` | 졸림 | 수면 부족, 칭얼거림 |
| `DISCOMFORT` | 불편함 | 기저귀, 옷, 자세 |
| `BURP` | 트림 필요 | 수유 직후 가스 |
| `PAIN` | 통증 | 배앓이 등 (의료적 신호) |
| `UNKNOWN` | 모르겠음 | 정정 시 사용 — 학습에서 무시 |

### 음향 feature

| 코드 | UI 한국어 | 단위 | 정의 / 의미 |
|------|----------|------|------------|
| `pitchMeanHz` | 음 높이 | Hz | 평균 기저주파수(F0). 600+ → 통증 신호 |
| `pitchStdHz` | 음 변동 | Hz | F0 표준편차. 120+ → 음정 변화 큼 |
| `pitchMaxHz` | (내부) | Hz | F0 최대값 |
| `voicedRatio` | (내부) | 0–1 | 유성음 프레임 비율 |
| `zcrMean` | (내부) | 0–1 | Zero Crossing Rate. 잡음성 척도 |
| `rhythmicity` | 리듬성 | 0–1 | RMS envelope 자기상관 피크. 0.45+ → 배고픔 패턴 |
| `cryConfidenceAvg` | 울음 강도 | 0–1 | Apple SoundAnalysis "infant_cry" 평균 |
| `cryConfidenceMax` | (내부) | 0–1 | 동상 최대 |
| `avgVolumeDb` | 평균 음량 | dBFS | 0이 최대, 음수만. -25 이상 = 큰 울음 |
| `peakVolumeDb` | (내부) | dBFS | 동상 최대 |

### 학습 단계 임계값

| 단계 | 확정 샘플 수 | 분류 동작 |
|------|--------------|----------|
| `HEURISTIC` | 0 ~ 19 | 컨텍스트 prior + 음향 규칙만 |
| `SIMILARITY` | 20 ~ 49 | 위 + 과거 확정 샘플과의 euclidean 유사도 |
| `PERSONAL` | 50+ | 위와 동일 (Phase 2B에서 진짜 개인 모델로 확장 예정) |

## 시스템 / 인프라

| 한국어 | 영어 | 정의 |
|--------|------|------|
| 핫리로드 | hot reload | Metro가 JS 변경을 즉시 앱에 반영 |
| 네이티브 빌드 | native build | Xcode로 Swift/Pod 컴파일 (Metro로는 안 됨) |
| Expo 로컬 모듈 | local module | `modules/<name>/` 에 두는 자체 네이티브 모듈 |
| CNG | Continuous Native Generation | Expo가 `app.config.ts` 기준으로 `ios/` 재생성하는 방식 |
