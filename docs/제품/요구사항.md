# 요구사항 (PRD)

> **누가 봄**: PM / FE / BE / QA
> **언제 봄**: 새 기능 정의 / 인수 기준 합의 / QA 시나리오 작성
> **상태**: 초기 버전 (출시 전, 일부 기능은 stub)

기능을 "X 기능 만든다"가 아니라 **"X 사용자가 Y 상황에서 Z를 한다"** 로 표현.

## 우선순위 표기

- **P0**: 출시 차단 (없으면 출시 불가)
- **P1**: 출시 1차 (기본 사용 가능)
- **P2**: 출시 2차 (있으면 좋음)
- **P3**: 백로그

## P1 — 울음 분석 (현재 구현)

### Phase 1 (heuristic + 학습 스텁)
- [x] 마이크 권한 요청

- [x] 컨텍스트(마지막 수유/기저귀/수면 경과) + 음향 규칙으로 라벨 예측
- [x] 라벨 6종: HUNGER / TIRED / DISCOMFORT / BURP / PAIN / UNKNOWN

- [x] 학습 단계 표시 (HEURISTIC → SIMILARITY → PERSONAL)

### Phase 2A (음향 feature 확장)
- [x] 피치(F0) / ZCR / 리듬성 추가

**인수 기준**:
- 마이크 권한 거부 시 명확한 안내

## 비-기능 요구

- **응답 속도**: 기록 저장 < 500ms, 통계 fetch < 1s
- **오프라인**: 일단 안 됨. 온라인 전제. (P3에서 큐잉 고려)
- **iOS**: 최소 iOS 16. iPad 미지원 (다음에).
- **Android**: 빌드는 됨. 1차 출시는 iOS만. 울음 분석은 iOS 전용 (SoundAnalysis 의존).
- **프라이버시**: 음성/사진 미저장. PII는 닉네임/메모 정도. [`운영/보안.md`](../운영/보안.md)
