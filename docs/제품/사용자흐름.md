# 사용자 흐름

> **누가 봄**: PM / 디자이너 / FE / QA
> **언제 봄**: 새 화면 추가, 흐름 변경, 사용성 평가

핵심 시나리오 5개. 화면 단위가 아니라 **사용자가 하려는 일** 단위로 정리.

## 1. 첫 시작 — 가족 만들기

```
[앱 첫 실행]
  → 가족 정보 없음 감지 (FamilySetupScreen)
  → "새 가족 시작하기" / "초대 코드로 합류"
  ┌─ "새 가족 시작" 선택
  │   → 가족 생성 → 초대 코드 받음
  │   → 아기 등록 (이름/생일/성별/출생체중·키)
  │   → 메인 탭으로 이동
  └─ "초대 코드로 합류" 선택
      → 8자리 코드 입력
      → 같은 가족의 데이터 보임
      → 메인 탭
```

**막히는 지점**:
- 초대 코드를 어디서 받는지 모름 → 첫 가족에서 BabyProfile 탭 → "가족 초대 코드" 카드
- 두 명이 동시에 새 가족 만들면 따로 놀게 됨 → 한 명이 먼저 만들고 코드 공유 필수

## 2. 일상 기록 — 수유

```
[홈 화면]
  → 빠른 기록 카드 "수유 +"
  → 모달: 양(ml) 또는 시간(분), 모유/분유 선택
  → 저장
  → 홈에 "다음 수유 예상: 14:30" 자동 표시
  → 14:25 (5분 전) 로컬 알림 발사
```

**대안 경로**: 기록 탭 → 수유 → 새로 추가 (더 자세히 입력하고 싶을 때)

## 3. 울음 분석 — 첫 사용

```
[울음 탭]
  → "분석 시작" 탭
  → 마이크 권한 요청 (첫 사용 시)
  ┌─ 권한 허용
  │   → 5초 카운트다운, "듣는 중..."
  │   → "분석 중..." (서버 응답 대기, ~1s)
  │   → 결과 화면:
  │       ┌─ 학습 단계 배너 (예: "신뢰도 학습 중 · 확인 0회")
  │       ├─ 가장 가능성 높은 이유 (큰 카드)
  │       ├─ 분석 디테일 (피치/리듬/음량)
  │       ├─ 다른 가능성 (작은 바)
  │       └─ "맞았어요" / "다른 이유예요" 버튼
  │   → "다른 이유예요" → 라벨 6종 모달 → 선택
  │   → 확정됨 표시
  └─ 권한 거부
      → 안내 + 설정으로 보내는 링크 (P2)
```

**중요**: "맞았어요"든 "다른 이유"든 사용자가 라벨 확정해야 학습이 진행됨. 이걸
유도하는 게 UX의 핵심.

## 4. 학습 진행 확인

```
[울음 탭]
  → "분석 기록 보기" 링크
  → CryHistoryScreen
      ┌─ 학습 단계 배너 (현재 단계 + 다음 단계까지 N회)
      ├─ 통계 카드 (총 분석 / 확인 완료 / 적중률)
      └─ 시간순 카드:
          - 적중 (녹색 배지) — 예측이 맞았던 경우
          - 정정됨 (오렌지) — 사용자가 다른 라벨 선택
          - 미확인 (회색) — 아직 라벨 안 함
  → 미확인 카드 탭 → 정정 모달 → 확정 → 통계 즉시 갱신
```

## 5. 가족 합류 (두 번째 기기)

```
[새 폰에서 앱 첫 실행]
  → "초대 코드로 합류" 탭
  → 코드 입력 (대소문자 구분 없음)
  → 같은 아기 데이터 즉시 표시
  → 둘이 같은 시간에 다른 기록 추가해도 충돌 없음 (서버 시간 기준)
```

**알려진 한계**: 동시에 같은 수유를 두 사람이 기록하면 두 건이 됨. 중복 검출 P3.

## 6. 산후조리원 등 임시 보호자

(P3 계획)
```
[제한 권한 코드] — 별도 코드 종류
  → "보기만 가능" 또는 "기록만 가능, 삭제 불가"
  → 기간 만료 자동 회수
```
